## 11. 시스템

- 최선의 시스템 구조는 POJO로 구현되는 모듈화된 도메인 영역이다.

 

> 시스템 제작과 사용을 분리하라
- Factory 패턴의 사용
- 의존성 주입(DI)의 사용
- 횡단 관심사를 묶는 AOP
    + Proxy 패턴 사용
    

> 의사결정을 최적화 할 것
- 관심사를 모듈로 분리한  POJO 시스템은 기민함을 제공하고 의사결정의 최적화를 이끈다.
- 시스템은 도메인 특화 언어(DSL)가 필요하다. DSL은 의사결정의 속도를 줄여준다.


> 시스템 수준에서 깨끗한 코드를 유지하는 방법
- 깨끗한 아키텍쳐의 설계
- 모든 추상화 단계에서의 명확한 의도 표현
- 각 구현 관심사의 분리
